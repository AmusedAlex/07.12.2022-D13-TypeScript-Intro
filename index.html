TypeScript Intro

TypeScript is a Superset of JavaScript

It means it can everything what JavaScript can PLUS more features


Main Feature: Static Typing

In JavaScript you can declare a variable and later redeclare it with another type:

let name = "Alexander"
name = 100 ✅

In TypeScript you chose a type for a variable, which cannot be changed by assigning. This helps us you detect errors before running the code

let name: string = "Alexander"
name = 100 // ❌ Does not work
name = "Abiodun" // ✅ this is called Type Inference. Once a type of variable is declared, it cannot be changed


Compiler

TypeScript cannot be read by the Browser so therefore we need a compiler to translate it into JavaScript.
Steps to follow:
1. create an index.html file
2. insert a <script src="index.js"> tag with the source to index.js
3. create your index.ts file
4. Open the terminal
5. type "tsc index.ts" for one-time compiling the index.ts file or "tsc index.ts -w" for ongoing compiling when saving the file


The Any Type

In TypeScript you can also use redeclarable types, using any. So later on you can change the type to whatever you want. Exactly like in JavaScript.

let anyVariable: any = "string"
anyVariable = 5 ✅
anyVariable = true ✅
anyVariable = "hahahhahahaha" ✅

The Union Type

In TypeScript you are able to use only a few types for a varibale. Others then configured will not be alowed.

let year: string | number
year = 2022 ✅
year = "2022" ✅
year = true ❌

Tuples

In TypeScript you can also declare specific and multiple types for values inside an array. The Order must be observed.

let arrTup: [string, number] = ["string", 5]
arrTup = ["hi", 100] ✅
arrTup = [100, "hi"] ❌

Interfaces

Interfaces are syntactical contracts that an entity (ex. Object) should conform to. In other words: An Interface defines the sytax that any entity must adhere to. Interfaces define properties, methods and events, which also can be the members of other interfaces.

interface PersonInterface {
name: string
age: number
workPermission: boolean
}

let mike: PersonInterface = {
name: "Mike",
age: 34,
workPermission: true,
} ✅

let mike: PersonInterface = {
name: "Mike",
age: "34",
workPermission: "yes",
} ❌ not allowed because the PersonInterface was declared with other types

Generics

Generic allow creating "type variables" which can be used to create classes, functions and type aliases that don't need to explicitly define the types that the use. Generics makes it easier to write reusable code.

interface Book<T> {
id: number
name: string
data: T
}

let aBook: Book<string> = {
id: 28734682734,
name: "Harry Potter and the Stoned Old Man",
data: "no information available", ✅
}

let otherBook: Book<string[]> = {
id: 28734682734,
name: "Harry Potter and the Stoned Old Man",
data: ["review1", review2], ✅
}
